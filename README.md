# عُدّة أخصائي سلامة الغذاء (Food Safety Specialist Toolkit)

أدوات مجانية عملية لأخصائيي سلامة الغذاء في الوطن العربي — تعمل بدون إنترنت، بدون حسابات، وبدون أي تكلفة. هذا المشروع صدقة جارية ولن يتحول لمدفوع أبداً.

## المبادئ الأساسية

- HTML + CSS + JavaScript خام فقط — بدون React أو أي إطار عمل أو أدوات بناء (build tools).
- لا يوجد خادم أو قاعدة بيانات أو تسجيل دخول. كل شيء يعمل داخل المتصفح.
- بياناتك (السجلات المحفوظة، اللغة، الثيم) تُخزَّن في `localStorage` على جهازك فقط، ولا تُرسل لأي مكان.
- يعمل بدون إنترنت بعد أول زيارة بفضل ملف Service Worker (`sw.js`) وملف `manifest.json` (PWA)، ويمكن تثبيته كتطبيق على الجوال أو الحاسوب (زر "تثبيت التطبيق" يظهر في الرأس عند توفر الإمكانية في متصفحك).
- نسخ احتياطي كامل: من الصفحة الرئيسية يمكنك تصدير كل بياناتك المحفوظة كملف JSON واحد، واستيرادها لاحقاً أو على جهاز آخر — لأن كل شيء محلي فقط ولا يوجد حساب سحابي.

## الأدوات المتوفرة

1. **حاسبة المطهرات** (`tools/sanitizer.html`) — تحضير محاليل تطهير بالتركيز الصحيح، فحص محاليل جاهزة، مدى مرجعي وزمن تلامس، وسجل تحضيرات قابل للطباعة والتصدير.
2. **دليل درجات الحرارة** (`tools/temperature.html`) — دليل مرجعي بحث/تصفية لدرجات الطبخ والتخزين والاستلام والتبريد السريع، وفاحص قراءة فوري بحكم وإجراء تصحيحي.
3. **شجرة قرار HACCP** (`tools/haccp.html`) — معالج تفاعلي بأسئلة الكودكس الأربعة لتحديد نقاط التحكم الحرجة (CCP)، مع مسار القرار وسجل تقييمات.
4. **مصفوفة المخاطر** (`tools/risk.html`) — تقييم احتمالية × شدة لكل خطر، خريطة حرارية مرجعية 5×5، وسجل تقييمات.
5. **معالج خطة HACCP** (`tools/haccp-plan.html`) — بناء خطة HACCP كاملة (بيانات المنتج والفريق + خطوات العملية بكل أعمدة المبادئ السبعة)، مع **استيراد تلقائي لنقاط التحكم الحرجة المحفوظة من أداة شجرة القرار** (الأداتان تتشاركان البيانات عبر `localStorage` لأنهما على نفس الأصل/origin)، وطباعة الخطة كاملة كمستند واحد.

## هيكل المشروع

```
/index.html                → الصفحة الرئيسية (بوابة الأدوات)
/manifest.json             → إعدادات تثبيت التطبيق (PWA)
/sw.js                     → عامل الخدمة للعمل بدون إنترنت
/assets/style.css          → التنسيق الموحد لكل الصفحات
/assets/app.js             → أدوات مشتركة: اللغة، الثيم، الرأس/التذييل، الطباعة، التخزين المحلي، النسخ الاحتياطي، تثبيت PWA
/assets/i18n.js            → كل نصوص الواجهة بالعربية والإنجليزية
/tools/sanitizer.html      → حاسبة المطهرات
/tools/temperature.html    → دليل درجات الحرارة
/tools/haccp.html          → شجرة قرار HACCP
/tools/risk.html           → مصفوفة المخاطر
/tools/haccp-plan.html     → معالج خطة HACCP
/icons/                    → أيقونات التطبيق (PWA)
```

## كيف تنشر الموقع على GitHub Pages (خطوة بخطوة)

1. تأكد أن الكود موجود في مستودع (repository) على GitHub الخاص بك.
2. من صفحة المستودع، افتح **Settings** (الإعدادات).
3. من القائمة الجانبية اختر **Pages**.
4. تحت **Build and deployment**، اختر المصدر (Source): **Deploy from a branch**.
5. اختر الفرع (Branch) الذي يحتوي على الملفات (مثلاً `main`) والمجلد `/ (root)`.
6. اضغط **Save**.
7. انتظر دقيقة أو اثنتين، سيظهر لك رابط الموقع أعلى الصفحة بصيغة:
   `https://username.github.io/repository-name/`
8. افتح الرابط — الموقع يعمل الآن مجاناً وللأبد على خوادم GitHub.

**بديل سريع لتجربة الموقع محلياً بدون رفعه:** افتح ملف `index.html` مباشرة في أي متصفح (Chrome/Firefox/Safari)، الموقع مصمم ليعمل حتى بفتح الملف مباشرة (`file://`) لأنه لا يحتاج خادماً.

> ملاحظة: خدمة Service Worker (العمل بدون إنترنت) تعمل فقط عند تصفح الموقع عبر `https://` أو `http://localhost`، وليس عند فتح الملف مباشرة بصيغة `file://` — هذا سلوك أمان قياسي في كل المتصفحات ولا علاقة له بكود المشروع.

## كيف تضيف أداة جديدة

المعمارية مصممة بحيث تكون إضافة أداة جديدة = **ملف HTML واحد + سطر واحد في السجل**. لا شيء آخر يتغير.

### الخطوات

1. **أنشئ صفحة الأداة**: انسخ `tools/temperature.html` كنقطة بداية (لأنه أبسط الأمثلة) وأعد تسميته، مثلاً `tools/traceability.html`. عدّل:
   - الروابط لملفات `../assets/style.css` و`../assets/i18n.js` و`../assets/app.js` (تبقى كما هي إن كان الملف داخل `/tools`).
   - استدعِ `initApp({ basePath: "../", activeToolId: "traceability" })` في نهاية الملف.

2. **أضف نصوص الأداة إلى `assets/i18n.js`**: أضف مفتاحاً جديداً (مثلاً `traceability`) داخل كائنَي `ar` و`en` يحوي كل النصوص التي ستستخدمها الأداة (العناوين، الأزرار، رسائل النتائج والتحذيرات). لا تكتب أي نص عربي أو إنجليزي مباشرة داخل كود الأداة — استخدم دائماً `t("traceability.مفتاح.ما")`.

3. **أضف الأداة إلى السجل** في `assets/app.js`، داخل `TOOLS_REGISTRY`:
   ```js
   { id: "traceability", icon: "🔍", href: "tools/traceability.html", available: true }
   ```
   وأضف اسم الأداة ووصفها في `assets/i18n.js` تحت `home.tools.traceability.name` و`home.tools.traceability.desc` بالعربية والإنجليزية.

4. **أضف الصفحة إلى قائمة التخزين المؤقت** في `sw.js` داخل `PRECACHE_URLS`، وارفع رقم `CACHE_VERSION` حتى يتحدّث المستخدمون الحاليون بالنسخة الجديدة.

5. **اكتب منطق الأداة** كدالة JavaScript صريحة ومقروءة داخل `<script>` في نفس ملف الأداة (وليس في `app.js` المشترك). اتبع الاصطلاحات المستخدمة في الأداتين الحاليتين:
   - كل قيمة/حد رقمي يجب أن يكون في متغير أو كائن بيانات واضح المعنى (لا أرقام سحرية داخل الشروط).
   - تحقق من صحة كل حقل إدخال باستخدام `parseNumberField()` من `app.js`.
   - أضف زر "طباعة كسجل" يستخدم `printRecordSheet()` من `app.js` لإخراج ورقة سجل جاهزة للطباعة والتوقيع.
   - إن أردت حفظ سجلات في `localStorage`، أضف مفتاحاً جديداً في `STORAGE_KEYS` بداخل `app.js`.

هذا كل شيء — الرأس، التذييل، تبديل اللغة، تبديل الثيم، والطباعة تعمل تلقائياً في أي صفحة تستدعي `initApp()`.

## الترخيص والغاية

هذا المشروع مجاني بالكامل، صدقة جارية لخدمة مجتمع سلامة الغذاء العربي. يمكن لأي شخص نسخه، تعديله، ونشره بحرية.

**تنويه مهم:** هذه الأدوات تساعد في اتخاذ القرار الفني ولا تُغني إطلاقاً عن الحكم المهني المختص أو الاشتراطات الرقابية الرسمية في بلدك أو منشأتك. تحقق دائماً من متطلبات جهتك الرقابية.
